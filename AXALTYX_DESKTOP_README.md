## 1. 产品概述

### 1.1 产品定位

```
AxaltyX = SPSS功能 × 3倍速度 × 现代UI × 本地优先
```

| 维度 | 描述 |
|------|------|
| **目标用户** | 本科生、研究生、科研人员、问卷与数据分析用户 |
| **产品定位** | 专业统计工具，功能 ≥ SPSS 标准版，速度 ≥ SPSS 3倍 |
| **使用模式** | 本地离线优先，可选字节账号登录同步配置 |
| **设计风格** | 现代简洁、字节设计语言（Arco Design / ByteDesign） |

### 1.2 核心价值主张

| 特性 | 说明 |
|------|------|
| **功能全面** | 覆盖SPSS标准版所有功能，新增高级分析方法 |
| **性能卓越** | 3倍SPSS速度，大数据集秒级响应 |
| **隐私安全** | 完全本地处理，数据不离开用户设备 |
| **现代UI** | 字节跳动Arco Design，简洁高效 |
| **易于使用** | 智能向导，拖拽式操作，结果自动解释 |

### 2.2 技术栈选择

| 层级 | 技术选型 | 选型理由 |
|------|---------|---------|
| **桌面框架** | Electron + React | 跨平台、成熟生态、字节团队熟悉 |
| **UI组件库** | Arco Design | 字节跳动官方设计语言、专业企业级 |
| **状态管理** | Zustand | 轻量级、高性能、API简洁 |
| **计算引擎** | Python + AxaltyX包 | 已开发核心库、丰富统计生态 |
| **性能优化** | Numba + 多进程 | JIT编译、并行计算、3倍速度目标 |
| **打包分发** | electron-builder | Windows/macOS/Linux安装包 |
| **数据可视化** | ECharts + Plotly | 专业统计图表、交互式可视化 |
| **离线优先** | IndexedDB + 本地文件 | 数据安全、无需联网 |

## 5. 性能优化

### 5.1 3倍速度目标实现路径

```
SPSS基准性能
    ↓
[1] 算法优化 → 1.5倍
    - 使用Numba JIT编译
    - 优化统计算法实现
    - 避免Python层面循环
    
    ↓
[2] 并行计算 → 2.0倍
    - 多核CPU并行处理
    - 任务级并行调度
    - 向量化操作
    
    ↓
[3] 内存优化 → 2.5倍
    - 高效数据结构
    - 内存映射大数据
    - 增量计算模式
    
    ↓
[4] 缓存智能 → 3.0倍
    - 分析结果缓存
    - 中间计算重用
    - 智能预加载
```

### 5.2 具体优化技术

| 技术 | 应用场景 | 预期提升 |
|------|----------|---------|
| **Numba JIT** | 核心统计计算 | 2-10倍 |
| **多进程** | 独立分析任务 | 2-4倍 (取决于核心数) |
| **NumPy向量化** | 数据转换操作 | 10-100倍 |
| **内存映射** | 1GB+大数据集 | 内存占用降80% |
| **结果缓存** | 重复分析 | 秒级响应 |
